# Feature Brief (PM Template)

Use this for backlog items. Keep answers short and specific.

## 1) Goal
- One sentence: what user value are we delivering?

## 2) Scope (In / Out)
### In Scope
- Bullet list.

### Out of Scope
- Bullet list.

## 3) User Stories
- As a <role>, I want <capability> so that <benefit>.

## 4) Acceptance Criteria
- Pass/fail bullets only.

## 5) UX / UI Notes
- Layout, states, and interactions (if applicable).

## 6) Dependencies / Stakeholders
- External services, teams, or approvals needed.

## 7) Delivery Notes
- Deadline, rollout, or priority notes.
