# Feature Brief -> Feature Spec Mapping

Use this checklist to expand a PM brief into the full engineering spec.

## Mapping
- Goal -> Feature Spec "Goal / Outcome"
- Scope -> Feature Spec "Scope (In / Out)"
- User Stories -> Feature Spec "User Stories"
- Acceptance Criteria -> Feature Spec "Acceptance Criteria"
- UX / UI Notes -> Feature Spec "UX / UI Notes"
- Dependencies / Stakeholders -> Feature Spec "Dependencies / External Services"
- Delivery Notes -> Feature Spec "Delivery Notes"

## Spec Completion Checklist (Engineering Adds)
- Data/API contracts (frontend inputs, backend endpoints, response shapes).
- Validation rules and error handling.
- Permissions/auth requirements.
- Analytics/logging requirements.
- Tests required (unit/integration/e2e).
- Migrations/backfill needs.
- Edge cases/constraints.
